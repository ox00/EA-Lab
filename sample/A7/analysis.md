# A7 选题分析与课题模板（GVGP & GVGAI）

## 1. 业务背景（具体到可落地场景）
A7 的核心目标是“通用游戏智能体”：在见过与未见过游戏/关卡上都保持竞争力。相比 A6 更强调泛化能力，而非单环境高分。GVGP/GVGAI 对应现实里的“规则变化下快速适应”问题。

典型可映射场景：

1. 规则频繁变化的平台决策系统
- 场景：营销策略、广告投放、流量分发策略迭代频繁。
- 诉求：模型面对新规则仍保持有效，不依赖大规模重训。

2. 新任务快速上线的自动化系统
- 场景：不同仓库布局、不同机器人任务模板。
- 诉求：策略跨场景迁移能力强，冷启动成本低。

3. 研究型通用智能验证
- 场景：统一 benchmark 比较 agent 的跨任务能力。
- 诉求：建立“泛化能力”而非“记忆能力”的评测框架。

核心点：A7 是“泛化导向”的高学术挑战选题，业务价值在于方法可迁移性，而不是单点性能最优。

## 2. 业务价值（可量化）
1. 泛化与迁移价值
- 减少每个新任务从零训练成本。
- 提升新场景冷启动成功率。

2. 研发效率价值
- 统一模型可覆盖多任务，降低模型体系复杂度。
- 缩短模型适配新规则的迭代周期。

3. 战略与学术价值
- 对“通用决策能力”有高辨识度，论文与答辩亮点明显。
- 可形成标准化的 seen/unseen 评测资产。

可落地量化目标示例：
- unseen 任务平均得分提升 10%-25%。
- 达到目标性能所需适应步数减少 20%-40%。
- seen 到 unseen 的性能回撤降低 15%-30%。

## 3. 关键难点（项目风险点）
1. 泛化难度高
- 训练分布与测试分布偏移显著，单任务技巧难迁移。

2. 评测协议严格
- 若 seen/unseen 划分不严谨，容易出现数据泄漏。
- 需要固定评估协议，否则结论不可信。

3. 表征学习与策略共享复杂
- 如何共享跨游戏特征同时保留任务差异是难点。
- 多任务训练容易负迁移。

4. 训练成本高
- 环境多、实验多、种子多，计算开销显著增加。

5. 结果波动大
- RL 方法在跨任务场景中方差更大，统计显著性要求更高。

## 4. 落地需要的主要资源
1. 数据与环境资源
- GVGAI Gym 或同类多游戏环境。
- 需要预先定义 seen/unseen 游戏集合。
- 日志：每任务回报、适应曲线、失败案例。

2. 工具与技术栈
- RL 框架：Stable-Baselines3 / RLlib / CleanRL。
- 表征与迁移：共享 backbone、多头策略、元学习（可选）。
- 评估脚本：统一评分、统一随机种子、统一预算。

3. 算力资源
- 推荐 GPU（至少 1 张）+ 多进程并行环境。
- CPU 可做小规模验证，但难支撑高质量泛化实验。

4. 人力与分工（4人组建议）
- 1 人：环境构建与 seen/unseen 划分。
- 1 人：策略模型与训练流程。
- 1 人：泛化评估与统计检验。
- 1 人：可视化、报告叙事与复现实验打包。

5. 工程保障资源
- 严格的实验协议文档（防数据泄漏）。
- 自动化评估流水线（按任务批量评测）。
- 模型版本管理与 checkpoint 追踪。

## 5. 课题模板（可直接选）
### 模板 A7-T1：跨游戏共享表征的多任务策略学习
1. 问题定义
- 输入：多个游戏任务的状态观测。
- 输出：共享 backbone + 任务特定策略头。
- 目标：提升 unseen 任务性能并降低性能回撤。

2. 数据与基线
- 环境：GVGAI 任务集（seen/unseen 拆分）。
- 基线：单任务 PPO、共享 backbone PPO、随机策略。

3. 评价指标
- 泛化指标：unseen 平均分、性能回撤率。
- 训练指标：样本效率、收敛速度。
- 稳定性指标：多种子均值与方差。

4. 预期产出
- 给出“共享程度 vs 泛化效果”的对比结论。
- 提供多任务训练配置建议。

### 模板 A7-T2：少样本适应的元策略（Meta-RL）
1. 问题定义
- 输入：历史任务集 + 新任务少量交互数据。
- 输出：可快速适应的初始化策略。
- 目标：减少新任务适应步数并保持最终性能。

2. 数据与基线
- 环境：多任务游戏集合。
- 基线：从零训练 PPO、微调策略、元学习策略。

3. 评价指标
- 适应速度：达到阈值得分所需步数。
- 最终性能：适应后得分。
- 成本指标：总训练时间与算力消耗。

4. 预期产出
- 证明少样本适应在新游戏上的效率优势。
- 给出“何时值得用 Meta-RL”的阈值建议。

### 模板 A7-T3：世界模型辅助的 GVGAI 泛化控制
1. 问题定义
- 输入：环境交互轨迹。
- 输出：世界模型 + 规划/策略联合决策。
- 目标：提升样本效率与 unseen 任务表现。

2. 数据与基线
- 环境：GVGAI + 控制变量扰动任务。
- 基线：model-free PPO/DQN、世界模型策略。

3. 评价指标
- 样本效率、unseen 得分、推理开销。
- 鲁棒性：扰动下性能衰减率。

4. 预期产出
- 给出 model-based 与 model-free 的边界条件。
- 形成可复用的泛化评估流程。

## 6. 课程内实施建议（按 case study 节奏）
1. 第一阶段（first report）
- 严格定义 seen/unseen 任务划分并写入协议。
- 选择 1 条主线（共享表征或少样本适应），避免多线并行。
- baseline 至少 2 个：单任务 + 多任务共享。

2. 第二阶段（final report 前）
- 做多随机种子与统计显著性检验。
- 加入 ablation：去掉共享层/去掉元学习模块。
- 输出泛化雷达图：seen、unseen、样本效率、稳定性。

## 7. 风险与规避
1. 风险：实验规模过大导致周期失控。
- 规避：限制任务数量（如 4-6 个 seen + 2-3 个 unseen）。

2. 风险：泛化结论不可信（协议不严格）。
- 规避：固定任务划分、预算和随机种子，并公开评估脚本。

3. 风险：模型复杂但收益不明显。
- 规避：先建立强基线，设置最小收益门槛（如 unseen 提升 >= 10%）。

4. 风险：答辩难以解释“为何有效”。
- 规避：加入表征可视化与失败案例分析，提升可解释性。

## 8. 课程适配评分（用于 Pipeline 比较）
评分标准：1-5 分，5 分最好。

1. 实施难度：2/5（泛化任务设计与评估复杂度高）
2. 数据可得性：4/5（环境可得，但协议搭建成本高）
3. 算力友好度：2/5（高质量实验通常依赖 GPU 与并行）
4. 创新空间：5/5（泛化、迁移、元学习方向创新度高）
5. 报告与答辩展示性：4/5（有亮点，但解释门槛较高）
6. 课程周期可控性：2/5（范围稍大就容易超时）
7. 综合推荐度：3.3/5（高潜力、高风险）

## 9. 结论（选题定位）
A7 是“通用智能导向”的高挑战选题，学术亮点强，但执行风险明显高于 A2/A3。若团队具备 RL 与实验工程经验、并有稳定 GPU 资源，可作为冲刺型方向；若优先稳健交付，建议将 A7 作为扩展实验而非主线。
