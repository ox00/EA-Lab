# A4 选题分析与课题模板（Dynamic Material Handling, DMH）

## 1. 业务背景（具体到可落地场景）
DMH 的核心是“动态到达任务 + 实时资源分配”，典型于制造与物流现场的在线调度问题。课程 A4 强调在实时约束下最小化 makespan 与 tardiness，和现实业务高度贴合。

1. 智能制造车间（AGV/叉车调度）
- 任务：原料补给、半成品转运、工位间配送。
- 冲突：及时性 vs 路径拥堵 vs 设备利用率。

2. 智能仓储与分拨中心
- 任务：上架、拣选补货、库位转运、月台装卸。
- 冲突：吞吐量 vs 超时订单 vs 机器人电量约束。

3. 机场/港口/医院后勤场景
- 任务：行李/集装箱/医疗物资的动态搬运。
- 冲突：应急优先级 vs 常规任务效率。

核心点：DMH 不是一次性离线优化，而是持续在线决策，算法要在不完整信息下快速给出可执行方案。

## 2. 业务价值（可量化）
1. 运营效率价值
- 降低任务等待时间，提升单位时间吞吐量。
- 缩短工单流转周期，减少关键工位堵塞。

2. 成本与产能价值
- 提升设备利用率，减少冗余运力投入。
- 在同等设备规模下承接更多任务。

3. 稳定性与韧性价值
- 对突发任务、设备故障、拥堵具备更强鲁棒性。
- 支持柔性制造和高波动业务场景。

可落地的量化目标示例：
- 平均任务延迟下降 15%-35%。
- 峰值拥堵时段吞吐提升 10%-25%。
- 逾期任务率下降 20% 以上。
- 设备利用率提升 8%-20%。

## 3. 关键难点（项目风险点）
1. 在线决策时效压力
- 决策窗口短，算法不能只追求最优值，还要保证实时可算。

2. 状态与约束高度动态
- 任务到达、设备状态、道路拥堵实时变化，导致环境非平稳。

3. 奖励与目标对齐困难
- RL 奖励若设计不当，容易出现局部最优或 reward hacking。

4. 泛化与鲁棒性不足
- 在训练场景表现好，不代表在新负载或故障场景依然有效。

5. 工程复杂度高
- 仿真环境、日志回放、在线评估链路都要打通，工作量大于静态优化。

## 4. 落地需要的主要资源
1. 数据资源
- 仿真数据：任务到达流、节点拓扑、运输时间矩阵、优先级标签。
- 真实数据（可选）：WMS/MES 历史任务日志、设备状态日志。
- 场景参数：高峰/低峰负载、故障概率、拥堵扰动。

2. 工具与技术栈
- 环境：DMH-GYM 或自建离散事件仿真环境。
- 算法：规则调度（FCFS/EDD）+ RL（DQN/PPO）+ 进化增强（ERL）。
- 实验框架：PyTorch、Stable-Baselines3/RLlib、日志可视化工具。

3. 计算资源
- 课程级最小可行：CPU 可跑规则基线 + 小规模 RL。
- 推荐：1 张中端 GPU 用于策略训练加速。
- 批量实验：多核并行 + checkpoint 续训能力。

4. 人力与分工（4人组建议）
- 1 人：场景建模与仿真器配置。
- 1 人：基线与 RL 策略实现。
- 1 人：鲁棒性实验与统计分析。
- 1 人：可视化、报告和复现打包。

5. 工程保障资源
- 实验配置版本化（场景参数、随机种子、模型版本）。
- 在线指标看板（延迟、吞吐、利用率、违例率）。
- 回放机制（失败案例复盘）。

## 5. 课题模板（可直接选）
### 模板 A4-T1：鲁棒 DMH 实时任务分配（规则 + RL）
1. 问题定义
- 输入：动态任务流、车辆/机器人状态、路径网络、时限约束。
- 输出：每个决策时刻的任务-设备分配与执行顺序。
- 目标：最小化平均 tardiness 与 makespan，控制冲突率。

2. 数据与基线
- 数据：DMH-GYM 场景 + 自定义扰动（故障、拥堵）。
- 基线：FCFS、EDD、最短处理时间优先 + PPO/DQN。

3. 评价指标
- 业务指标：平均延迟、逾期率、吞吐量、设备利用率。
- 算法指标：收敛步数、训练稳定性、推理时延。

4. 预期产出
- 在静态与扰动场景下的对比曲线。
- “何时采用 RL、何时回退规则法”的决策建议。

### 模板 A4-T2：进化强化学习优化约束下调度策略
1. 问题定义
- 输入：同上，附加约束（电量、安全距离、优先级 SLA）。
- 输出：策略参数与在线调度决策。
- 目标：提升收益同时降低约束违例率。

2. 数据与基线
- 数据：多负载强度场景集（低/中/高峰）。
- 基线：纯 RL、纯启发式、固定阈值策略。

3. 评价指标
- 业务指标：任务完成率、违例率、峰值时延。
- 算法指标：跨场景迁移性能、稳定性、训练开销。

4. 预期产出
- 证明 ERL 在复杂约束场景下的优势边界。
- 输出可复用的策略训练流程模板。

### 模板 A4-T3：云化物料搬运与算力资源协同调度
1. 问题定义
- 输入：搬运任务 + 边云计算资源状态。
- 输出：任务执行分配 + 计算资源分配。
- 目标：联合最小化任务延迟与计算成本。

2. 数据与基线
- 数据：仿真任务流 + 资源池日志。
- 基线：分离式调度（先搬运后算力）与联合优化策略。

3. 评价指标
- 业务指标：端到端延迟、单位任务成本、SLA 违约率。
- 系统指标：资源利用率、调度开销。

4. 预期产出
- 联合调度相对分离调度的增益报告。
- 可扩展到智能工厂云边协同的原型框架。

## 6. 课程内实施建议（按 case study 节奏）
1. 第一阶段（first report）
- 锁定单一主场景（建议 AGV 车间搬运）。
- 明确 KPI：平均延迟、吞吐、违例率。
- 固定 baseline：2 个规则法 + 1 个 RL。

2. 第二阶段（final report 前）
- 增加扰动实验：高峰任务、设备故障、路径阻塞。
- 做多随机种子与多场景复验。
- 给出策略部署建议：在线推理时延与回退机制。

## 7. 风险与规避
1. 风险：训练不收敛或波动大。
- 规避：先跑规则基线，分阶段增加 RL 复杂度；保留 early-stop 与回退策略。

2. 风险：仿真过于理想化，业务说服力弱。
- 规避：注入扰动与异常场景，报告鲁棒性指标。

3. 风险：算力不足导致实验跑不完。
- 规避：先小规模场景验证，再扩规模；控制超参数网格大小。

4. 风险：结论不可复现。
- 规避：固定种子、保存模型与配置、提供一键实验脚本。

## 8. 课程适配评分（用于 Pipeline 比较）
评分标准：1-5 分，5 分最好。

1. 实施难度：3/5（在线调度 + RL，复杂度较高）
2. 数据可得性：4/5（可依赖仿真，真实数据可选）
3. 算力友好度：3/5（可 CPU 起步，但高质量实验更依赖 GPU）
4. 创新空间：5/5（鲁棒调度、约束学习、ERL 结合空间大）
5. 报告与答辩展示性：4/5（曲线和回放展示较强）
6. 课程周期可控性：3/5（需要严格范围管理）
7. 综合推荐度：3.9/5（创新强、风险也较高）

## 9. 结论（选题定位）
A4 是“高业务贴合 + 高方法挑战”的中高风险选题。若团队有强化学习经验且能获得基本 GPU 资源，A4 可做出有亮点的成果；若目标是稳健交付，建议缩小范围为“单场景 + 规则法对比 + 轻量 RL”，避免过度扩展。
