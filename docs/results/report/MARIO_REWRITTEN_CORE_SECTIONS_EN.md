# Mario Level Generation via Hard-Constrained Multi-Objective Evolutionary Search with AI-Seeded Initialization

This file rewrites the three most visible report sections in a cleaner academic style while keeping the current project scope and evidence aligned with the repository.

## Abstract
Procedural content generation for platform games requires more than producing valid layouts. A useful generator must enforce playability, control difficulty, preserve structural variation, and expose the trade-offs among these goals. We present a Mario level generation pipeline built around an explicit segment-based genotype, deterministic decoding, hard feasibility constraints, and multi-objective evolutionary search. The search process uses NSGA-II to optimize difficulty error, structural diversity, and emptiness error, and it is later extended with family balance and difficulty curve error to shape higher-level level semantics. We also integrate an AI-seeded initialization path based on VGLC-derived segment sequences and an adapter layer that normalizes or repairs generated chromosomes before they enter the initial population. The experiments show that the hard-constrained search pipeline is stable, that semantic objectives materially change the shape of the Pareto frontier, and that repaired AI-seeded initialization becomes usable even when raw AI seeds are biased by the training distribution. The final system is presented through an interactive frontier browser with representative level renders, constraint-level reachability replay, and lite-physics replay, which turns optimization outputs into inspectable evidence rather than static screenshots.

## Introduction
Procedural content generation has long been used to reduce authoring cost and increase replayability in games, but for platformers such as Mario, generation quality is constrained by several competing requirements. A level should be feasible to traverse, yet it should not collapse into a trivial flat corridor. It should exhibit structural variation, yet still maintain readable pacing and manageable density. These requirements are not naturally captured by a single scalar score. They are better treated as a constrained multi-objective search problem.

This project studies Mario level generation from that perspective. The core design choice is to represent a level as a fixed-length chromosome of segment identifiers. Each chromosome is decoded deterministically into a tile grid, after which a hard feasibility gate checks whether the level contains valid start and goal placements, respects tile placement rules, keeps gaps within a jumpable range, and admits a reachable route under the same simplified movement model used by the constraint checker. This representation makes the search space explicit and keeps the interface between generation, evaluation, and rendering simple enough for reproducible experiments.

On top of this representation, we use evolutionary search to optimize several soft design objectives. The baseline configuration controls difficulty error, structural diversity, and emptiness error. Two semantic extensions are then added. Family balance discourages over-concentration in a single segment family and rewards more even structural composition, while difficulty curve error encourages the sequence of segment difficulty tiers to follow a target progression from easier to harder content. Because these objectives conflict, we use NSGA-II rather than collapsing them into a hand-tuned weighted sum. This lets the system retain multiple high-quality trade-off solutions and makes the final frontier more useful for inspection and presentation.

The project also keeps an AI line, but in the correct role. Instead of replacing the search engine, the AI component serves as an upstream initializer. We extract approximate segment chromosomes from VGLC Mario levels, train an LSTM sequence model on those chromosomes, and use its outputs to seed part of the initial population. An adapter layer trims invalid IDs, normalizes sequence length, and optionally applies feasibility-aware repair before the seeded chromosomes enter the EA pipeline. This means AI-seeded initialization is not a detached side experiment. It is a real integrated input path that can be evaluated against random initialization under the same downstream search process.

The strongest outcome of the project is therefore not a claim that AI alone generates superior Mario levels. The stronger contribution is that we build a controllable and presentation-ready PCG pipeline in which genotype design, hard constraints, objective design, and final visualization all work together. The browser-based frontier viewer and layered replay evidence make the results easier to interpret, and they help connect optimization metrics to visible level differences. This is particularly important in a course project setting, where clarity of design choices matters as much as raw generation output.

## Conclusion
This project establishes a complete Mario PCG pipeline centered on explicit representation, hard feasibility control, and multi-objective evolutionary search. The segment-based genotype and deterministic decoder define a clear search space. The constraint layer acts as a strict feasibility gate. NSGA-II then preserves a set of non-dominated solutions instead of forcing all design preferences into one score. As a result, the system can expose meaningful trade-offs among difficulty control, structural variation, density, family composition, and pacing.

The experimental results support three conclusions. First, the base search pipeline is stable: feasible populations are consistently obtained, which confirms that the genotype-decode-check-evaluate loop is working as intended. Second, semantic objectives matter. Family balance changes how segment families are distributed across a level, while difficulty curve error changes the pacing profile along the chromosome. These objectives do not simply improve one number; they change the shape of the frontier and the type of levels the search prefers. Third, AI-seeded initialization is a genuine integrated extension, but its usefulness depends on upstream data quality. Raw AI seeds inherit the skew of the training set, whereas adapter-level repair restores feasibility and brings the seeded mode closer to the random baseline.

The final contribution is the interpretation layer. The interactive frontier browser, representative renders, reachability replay, lite-physics replay, and exported action plans make the generated levels easier to inspect and defend. This matters because PCG results are otherwise difficult to explain through tables alone. The current system still has clear limits, especially in segment library expressiveness, AI seed quality, and the fidelity of the lightweight physics model. Even so, the project now stands as a coherent and extensible foundation: AI can be improved upstream, new objectives can be added without changing the search framework, and the final demonstration layer is strong enough to support report and presentation use.
