# A2 选题分析与课题模板（Fair Machine Learning, FairML）

## 1. 业务背景（具体到可落地场景）
FairML 适用于“模型预测会影响个人机会或资源分配”的高敏感业务。典型场景如下：

1. 信贷审批与授信额度
- 决策对象：通过/拒绝、授信额度、利率分层。
- 业务冲突：违约风险控制 vs 群体公平（性别、年龄、地区等）。

2. 招聘筛选与人才推荐
- 决策对象：简历筛选、面试排序、录用建议。
- 业务冲突：岗位匹配度 vs 机会公平与反歧视要求。

3. 保险风控与定价
- 决策对象：承保、保费、理赔审核优先级。
- 业务冲突：精算准确度 vs 敏感属性间的不公平差异。

4. 教育与公共服务分配
- 决策对象：奖学金、入学推荐、服务优先级。
- 业务冲突：有限资源效率分配 vs 社会公平目标。

课程 A2 给出 Adult、Bank 两套经典数据集与 Fairlearn 工具，数据和工具链成熟，适合在课程周期内完成“可复现 + 可解释”的研究。

## 2. 业务价值（可量化）
1. 合规与风险价值
- 降低歧视投诉与审计处罚风险。
- 提升模型“可审计性”，便于对外/对内合规说明。

2. 商业与品牌价值
- 在可控性能损失下提升弱势群体可达性，扩大潜在客群。
- 减少因算法偏见引发的品牌负面事件。

3. 运营与管理价值
- 建立“准确率-公平性-收益”三方权衡机制。
- 形成持续监控指标（如 EO gap、DP diff），支持版本治理。

可落地的量化目标示例：
- 公平性差异指标下降 20%-40%。
- AUC/F1 下降控制在 1%-3% 以内。
- 在同等审核成本下提升目标群体覆盖率。

## 3. 关键难点（项目风险点）
1. 公平定义选择难
- Demographic Parity、Equal Opportunity、Equalized Odds 的业务含义不同。
- 指标之间可能冲突，无法同时最优。

2. 历史偏差与标签偏差
- 训练数据可能反映历史不公平，不是“真实能力”。
- 标签（如是否违约）本身可能受制度偏差影响。

3. 性能-公平性折中难
- 强力去偏可能导致业务收益下降或误拒率上升。
- 需要业务方提供可接受的性能回撤阈值。

4. 上线治理难
- 训练后公平不代表线上公平，数据漂移会使偏差回弹。
- 需要监控、告警、重训练与人工复核策略。

## 4. 落地需要的主要资源
1. 数据资源
- 课程级：Adult、Bank Marketing（开箱可用）。
- 业务级：历史审批/招聘记录（需脱敏与合规审批）。
- 必备字段：特征、标签、敏感属性、时间戳。

2. 工具与技术栈
- 建模：scikit-learn / XGBoost。
- 公平工具：Fairlearn（课程推荐），可补充 AIF360。
- 实验管理：Jupyter + 固定随机种子 + 配置文件。

3. 计算资源
- CPU 即可满足课程实验。
- 不依赖 GPU，算力门槛低，适合快速迭代。

4. 人力与分工（4人组建议）
- 1 人：数据处理与偏差诊断。
- 1 人：基线模型与去偏方法实现。
- 1 人：评价体系与统计检验。
- 1 人：报告叙事与复现打包。

5. 合规与流程资源
- 敏感属性使用边界说明。
- 模型卡（Model Card）与公平性报告模板。
- 审核流程：异常样本人工复核规则。

## 5. 课题模板（可直接选）
### 模板 A2-T1：贷款审批的“准确率-公平性”双目标优化
1. 问题定义
- 输入：申请人特征、历史还款标签、敏感属性。
- 输出：通过概率与审批阈值策略。
- 目标：最大化 AUC/F1，最小化 EO gap 与 DP diff。

2. 数据与基线
- 数据：Adult +（可选）模拟信贷数据。
- 基线：Logistic Regression、XGBoost、未去偏模型。
- 去偏方法：reweighing、threshold optimization、Fairlearn post-processing。

3. 评价指标
- 预测性能：AUC、F1、Recall、Brier score。
- 公平指标：EO gap、DP diff、group TPR/FPR gap。
- 业务指标：通过率、坏账代理指标、人工复核率。

4. 预期产出
- 准确率-公平性 Pareto 曲线。
- 不同风险偏好下的阈值推荐表。

### 模板 A2-T2：招聘筛选中的公平重排序
1. 问题定义
- 输入：候选人特征与历史筛选结果。
- 输出：候选人排序/面试名单。
- 目标：在固定 Recall@K 下最小化群体通过率差异。

2. 数据与基线
- 数据：Bank + 自建招聘仿真数据。
- 基线：原始排序模型、群体独立阈值、re-ranking。

3. 评价指标
- 效果指标：Recall@K、NDCG@K。
- 公平指标：selection rate gap、EO gap。
- 运营指标：面试资源利用率、误筛率。

4. 预期产出
- 统一阈值 vs 分群阈值对比分析。
- 公平提升与招聘效率损失边界。

### 模板 A2-T3：多目标集成去偏（参考 FairEMOL 思路）
1. 问题定义
- 输入：结构化特征与敏感属性。
- 输出：模型集成权重与预测结果。
- 目标：联合优化性能与公平性（可扩展为多目标）。

2. 数据与基线
- 数据：Adult、Bank。
- 基线：单模型 + 常规后处理。

3. 评价指标
- HV/IGD（若采用多目标优化框架）。
- AUC/F1 + 公平指标组合。

4. 预期产出
- 证明多目标方法在不同公平约束下的稳健性。

## 6. 课程内实施建议（按 case study 节奏）
1. 第一阶段（first report）
- 明确场景：贷款或招聘二选一，避免题目过大。
- 固定公平定义：建议 EO gap + DP diff 两项。
- 锁定 baseline：未处理模型 + 2 种去偏方法。

2. 第二阶段（final report 前）
- 做参数网格与多随机种子重复。
- 形成“性能回撤阈值”讨论：例如 AUC 下降不超过 2%。
- 输出推荐策略：不同业务偏好下选哪个模型版本。

## 7. 风险与规避
1. 风险：只讲公平概念，实验证据不足。
- 规避：固定实验协议，报告平均值和标准差。

2. 风险：敏感属性处理不当导致结论不可信。
- 规避：明确哪些属性用于训练、评估、后处理，并做消融实验。

3. 风险：只提升公平但性能损失过大。
- 规避：设置业务可接受区间（如 AUC 回撤 <= 2%）。

4. 风险：难以说明上线可行性。
- 规避：提供监控指标、告警阈值、人工复核流程草案。
