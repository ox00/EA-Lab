# A3 选题分析与课题模板（Vehicle Routing Problems, VRPs）

## 1. 业务背景（具体到可落地场景）
VRP 是物流与现场服务的核心优化问题，适用于“多车辆、多站点、强约束”的路线与排班决策。课程 A3 给出的 3 个子题（CARP/VRPTW/SDVRP）都可直接对应真实业务：

1. CARP（Capacitated Arc Routing Problem）
- 典型业务：环卫清扫、道路巡检、除雪、管网巡查。
- 特征：服务对象是“路段/弧”，不是客户点；每条需服务弧有需求量。
- 约束：车辆容量限制、作业区覆盖完整性。

2. VRPTW（Vehicle Routing Problem with Time Windows）
- 典型业务：同城即配、冷链配送、上门安装/维修。
- 特征：客户必须在指定时间窗被服务，且涉及等待/服务时长。
- 约束：时窗硬约束、容量约束、司机工时。

3. SDVRP（Split Delivery VRP）
- 典型业务：高峰期大单配送、B2B 补货、仓配协同。
- 特征：同一客户需求可由多车拆分送达。
- 约束：总交付量满足客户需求，且每辆车仍需容量可行。

核心点：A3 属于“业务价值最直观”的选题，优化结果可直接转化为成本、准时率与运力利用率提升。

## 2. 业务价值（可量化）
1. 成本与效率价值
- 总里程/油耗降低，车辆空驶减少。
- 同车队规模下提升日处理订单量。

2. 服务质量价值
- 准时率提升，超时赔付与客诉下降。
- 可为 SLA 管理提供量化支撑（如 P95 到达时延）。

3. 运力管理价值
- 高峰期通过拆分送达缓解容量瓶颈。
- 支持“少增车甚至不增车”的增长策略。

可落地的量化目标示例：
- 总运输成本下降 8%-20%。
- 准时率提升 5%-15%。
- 车辆利用率提升 10%-25%。
- 高峰拒单率下降 20% 以上（SDVRP 场景）。

## 3. 关键难点（项目风险点）
1. 组合复杂度高
- 约束叠加后搜索空间爆炸，精确解法在中大规模上代价高。

2. 数据与现实偏差
- 基准集（如 Solomon）与真实城市交通差异明显。
- 若不引入动态拥堵/取消订单，结论可能偏乐观。

3. 静态优化到在线执行的鸿沟
- 实际订单会动态到达，离线最优路由可能快速失效。
- 需考虑重优化频率与实时性开销。

4. 多目标冲突
- 降低成本与提升准时率常冲突。
- 分单公平性、司机劳动规则等约束会进一步压缩可行空间。

## 4. 落地需要的主要资源
1. 数据资源
- 学术数据：Solomon（VRPTW）、CARP benchmark、SDVRPLIB。
- 业务化数据：订单时间戳、地理坐标、需求量、服务时长、时窗、车队信息。
- 扩展数据：路网/ETA 估计（可先用仿真替代）。

2. 工具与技术栈
- 建模与求解：OR-Tools、PyVRP、启发式/元启发式（GA/TS/ALNS）。
- 实验管理：固定随机种子、参数配置文件、批量实验脚本。
- 可视化：路径图、甘特图、成本-准时率权衡图。

3. 计算资源
- 课程级实验：CPU 中等即可。
- 多实例多种子：建议多核并行。
- GPU 通常非刚需。

4. 人力与分工（4人组建议）
- 1 人：数据清洗与实例构建。
- 1 人：建模与约束编码。
- 1 人：算法实现与调参。
- 1 人：实验分析、可视化与报告打包。

5. 工程化资源
- 结果复现脚本（从原始数据到最终表图）。
- 路由合法性检查器（容量/时窗/工时）。
- 指标计算模块（成本、准时率、利用率、稳定性）。

## 5. 课题模板（可直接选）
### 模板 A3-T1：VRPTW 的“低成本-高准时”联合优化
1. 问题定义
- 输入：客户点、需求、时窗、服务时长、车队容量。
- 输出：车辆路径与访问顺序。
- 目标：最小化总成本与迟到惩罚。

2. 数据与基线
- 数据：Solomon benchmark + 1 套自建城市配送数据。
- 基线：Clarke-Wright、OR-Tools、禁忌搜索/ALNS。

3. 评价指标
- 学术指标：可行率、解质量、运行时间。
- 业务指标：总成本、准时率、平均/95 分位迟到时长。

4. 预期产出
- 成本-准时率 trade-off 曲线。
- 在不同 SLA 约束下的推荐调度策略。

### 模板 A3-T2：SDVRP 在高峰期的容量缓解策略
1. 问题定义
- 输入：高峰订单、车辆容量、客户需求。
- 输出：允许拆分下的多车配送计划。
- 目标：最小化超时与拒单，同时控制运输成本。

2. 数据与基线
- 数据：SDVRPLIB + 高峰仿真订单。
- 基线：不拆分 VRP、经典 SDVRP 启发式。

3. 评价指标
- 业务指标：拒单率、超时率、车辆利用率、单位订单成本。
- 稳定性指标：多随机种子方差。

4. 预期产出
- 证明拆分策略在高峰工况下的增益边界。
- 给出“何时启用拆分”的业务规则。

### 模板 A3-T3：CARP 的城市作业车辆调度优化
1. 问题定义
- 输入：道路弧段需求、车辆容量、作业时段。
- 输出：车辆对弧段的服务序列。
- 目标：最小化总作业成本与任务完工时间。

2. 数据与基线
- 数据：CARP benchmark + 城市道路简化拓扑。
- 基线：路径扫描、遗传算法、局部搜索。

3. 评价指标
- 业务指标：总里程、总工时、覆盖完成率、超时作业比例。
- 算法指标：收敛速度、鲁棒性（扰动场景下性能保持）。

4. 预期产出
- 城市运维任务的路线优化模板。
- 不同车队规模下的资源配置建议。

## 6. 课程内实施建议（按 case study 节奏）
1. 第一阶段（first report）
- 三个子题中优先锁定一个（建议 VRPTW 或 SDVRP）。
- 固定 2 个 baseline + 1 个改进方向（如约束处理或邻域算子）。
- 明确业务 KPI：成本、准时率、利用率。

2. 第二阶段（final report 前）
- 完成多实例 + 多随机种子实验。
- 加入扰动测试（高峰需求、路网延迟）验证鲁棒性。
- 形成“选解策略”：不同业务偏好下推荐哪类方案。

## 7. 风险与规避
1. 风险：问题范围过大（同时做 CARP+VRPTW+SDVRP）。
- 规避：主攻一个子题，另一个作为扩展实验。

2. 风险：只在小规模实例有效，缺少说服力。
- 规避：覆盖小中大规模实例并报告计算时间。

3. 风险：指标只看里程，忽视服务质量。
- 规避：至少同时报告成本 + 准时率 + 稳定性。

4. 风险：代码可运行但不可复现。
- 规避：固定种子、记录参数、提供一键运行脚本。

## 8. 课程适配评分（用于 Pipeline 比较）
评分标准：1-5 分，5 分最好。

1. 实施难度：4/5（中等偏上，可通过成熟工具控制复杂度）
2. 数据可得性：5/5（公开 benchmark 完整）
3. 算力友好度：4/5（CPU 可做，批量实验需并行）
4. 创新空间：4/5（可在约束处理、算子设计、鲁棒性上创新）
5. 报告与答辩展示性：5/5（路径图与业务指标非常直观）
6. 课程周期可控性：4/5（范围控制得当可稳定交付）
7. 综合推荐度：4.3/5（高业务价值、强展示型选题）

## 9. 结论（选题定位）
A3 是“业务落地最直观、展示效果最好”的强应用型选题。若你们希望在答辩中清晰展示业务增益（降本、提准时、提利用率），A3 非常合适；建议优先选 VRPTW 或 SDVRP 作为主线，再用另一个子题做扩展，以平衡创新度与可交付性。
